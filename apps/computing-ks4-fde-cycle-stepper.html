How the cycle works (revision card)
  • Fetch: the address in the PC is copied to the MAR. RAM returns the instruction into the MDR, which is copied to the CIR. The PC then increments by 1, ready for the next instruction.
  • Decode: the control unit interprets the CIR: which operation, and which operand (address or value).
  • Execute: the ALU or control unit carries it out, for example loading from RAM into the ACC, or adding to the ACC.
  • RAM is volatile primary storage: it loses its contents when power is removed. ROM holds the boot program and is non-volatile. Secondary storage (SSD, hard disk) keeps programs when off.
