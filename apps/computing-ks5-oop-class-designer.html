        The big ideas of object-oriented programming (OCR H446)
        

          
In the object-oriented paradigm a program is modelled as a set of co-operating objects. A class is a template (a user-defined data type) that bundles together state and behaviour; an object is an instance created from that class.

          

            
ClassA blueprint defining the attributes and methods that its objects will have. CLASS Animal.

            
Object / instanceA specific thing built from a class, with its own values. rex = NEW Dog("Rex", 3).

            
Attribute (field)A variable that holds an object's state, e.g. name, age.

            
MethodA subroutine belonging to the class that defines behaviour, e.g. speak().

            
ConstructorA special method (NEW / __init__) that runs when an object is created, setting up its initial state.

            
EncapsulationBundling state and behaviour together and keeping fields private, exposing them only through public getters and setters. Protects data integrity and hides implementation.

            
Getter / setterPublic methods that read (getName()) or controlled-write (setAge()) a private field, so validation can be enforced.

            
InheritanceA subclass reuses and extends a superclass: Dog INHERITS Animal. Models an is-a relationship and avoids code duplication.

            
OverridingA subclass replaces an inherited method with its own version of the same name.

            
Polymorphism"Many forms": the same method call behaves differently depending on the object's actual class, resolved at run time.

            
Composition / aggregationComposition is a strong has-a (a Car owns its Engine; destroy the car and the engine goes too). Aggregation is a weaker has-a (a Team has Players that can outlive it).

          

          
Why OOP? Encapsulation protects data and localises change; inheritance and polymorphism cut duplication and let one routine work over many types; classes map cleanly onto real-world entities, which improves modularity, reuse, maintainability and team working on large systems.

        

      


      

        How to use this designer
        

          
1. Define a superclass such as Animal with fields name and age and a method speak. Press Save class to model.

          
2. Define Dog and set Inherits from to Animal. Add a speak method again to override it (e.g. says Woof).

          
3. Define Cat the same way (says Meow).

          
4. In Instantiate objects, create a Dog and a Cat, then in the object collection choose speak and press Send message. Each object answers in its own voice: polymorphism. The shortcut button Load Animal → Dog / Cat demo sets all of this up for you.
