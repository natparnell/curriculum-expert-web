Explainer: virtual memory, page faults and thrashing

Virtual memory is a technique that uses part of the secondary storage as if it were extra RAM, so programs can run even when physical RAM is full. The operating system divides memory into fixed-size blocks called pages.

When a program needs a page that is not currently in RAM, a page fault occurs and the OS must fetch that page from disk (a page-in), often swapping a less-used page out first (a page-out). Because secondary storage is far slower than RAM, each swap costs time.

If RAM is badly overcommitted the OS spends most of its time swapping pages in and out rather than doing useful work. This is called disk thrashing and it makes the computer feel very slow. The fix is more RAM, fewer open programs, or faster secondary storage (SSD over HDD). This is why more RAM generally improves performance.

Primary memory: RAM vs ROM Recap

RAM: Random Access Memory

  • Volatile: loses its contents when power is removed.
  • The computer's working memory: holds the OS, open programs and data in use.
  • Can be read from and written to; the CPU accesses it far faster than secondary storage.
  • More RAM means fewer page faults and less reliance on virtual memory.

ROM: Read Only Memory

  • Non-volatile: keeps its contents when power is removed.
  • Holds the bootstrap / BIOS: the start-up instructions that run when the machine is switched on.
  • The BIOS loads the operating system from secondary storage into RAM, then hands over control.
  • Normally read only, so its essential start-up code cannot be accidentally overwritten.

Exam-style questions Reveal answers

State what is meant by the term volatile and name one type of memory that is volatile. [2]
Volatile means the memory loses its contents when the power is switched off. RAM is volatile.
A computer has 8 GB of RAM. Explain what happens when the user opens more programs than will fit in RAM, and the effect on performance. [4]
When RAM is full the operating system uses virtual memory: it moves the least-used pages from RAM out to secondary storage (the page file). This frees RAM for the new program. When a swapped-out page is needed again a page fault occurs and the page must be read back from disk. Because secondary storage is much slower than RAM, lots of swapping (thrashing) makes the computer slow and unresponsive.
Give two reasons why a school might choose magnetic hard drives rather than solid state drives for backing up its file server. [2]
1. Magnetic drives have a much lower cost per GB, so large backups are cheaper. 2. They offer very high capacities, suitable for storing large volumes of data. (Backup speed matters less than for everyday use.)
An uncompressed bitmap image is 1000 pixels wide, 500 pixels tall, with a colour depth of 8 bits. Calculate its size in kilobytes. [3]
Bits = 1000 × 500 × 8 = 4,000,000 bits. Bytes = 4,000,000 ÷ 8 = 500,000 bytes. kB = 500,000 ÷ 1000 = 500 kB.
Explain why ROM is used to store the boot-up instructions rather than RAM. [2]
ROM is non-volatile, so the start-up (bootstrap/BIOS) instructions are still present when the computer is switched on. RAM is volatile and would be empty at power-on, so it could not hold the instructions needed to start the machine.
Describe one advantage and one disadvantage of optical storage compared with solid state storage. [2]
Advantage: optical discs are very cheap and highly portable, ideal for distributing films or software. Disadvantage: they have far lower capacity and much slower read/write speeds than SSDs, and are easily scratched.
