Explainer key ideas

Arrays: same type, indexed from zero

An array is a data structure that stores many values of the same data type under one identifier. Each value sits in a numbered slot called an element, and the slot number is the index.

  • Indexing from 0: the first element is scores[0], not scores[1]. An array of length 8 has valid indexes 0 to 7.
  • Write: scores[3] = 99 stores 99 at index 3, overwriting whatever was there.
  • Read: x = scores[3] retrieves the value at index 3.
  • Traverse: a for loop visits each index in turn, e.g. to find the largest value or to search.
2D arrays: rows and columns

A two-dimensional array stores a grid of values, addressed by two indexes: array[row][col] (OCR writes this as array[row,col]). Both count from 0.

Uses include a class register (pupils x days), a noughts-and-crosses board, an image as a grid of pixels, or a spreadsheet. To process every cell you use a nested loop: an outer loop over rows and an inner loop over columns.

Records: different types, named fields

A record groups related fields that can be different data types. A Student record might hold a name (String), an age (Integer) and a tutorGroup (String) together as one item.

You access a single field with dot notation: student.age. Records are often combined with arrays, for example an array of Student records to hold a whole class.

Array vs record: an array is many items of one type indexed by number; a record is one item with several named fields of possibly different types.
