Protocol Stack Builder

Encapsulation and de-encapsulation across the four-layer TCP/IP model

Teacher aim: Pupils model how data is wrapped with protocol headers as it moves down the TCP/IP stack at the sender, and unwrapped at the receiver, linking each layer to its protocols, addresses and job. Maps to GCSE Computer Science: networks, topologies, protocols and the internet.

1. Set up the transmission

Choose a message, then a direction. Sending wraps headers on (encapsulation); receiving strips them off (de-encapsulation).

Application Transport Internet Link

2. Move the data through the stack

Drag the data block onto the correct next layer. Keyboard: focus the block, press Enter or Space to move it to the highlighted layer.

Application layer HTTP · HTTPS · SMTP · IMAP · FTP · DNS
Creates the message in a format the app understands.
Transport layer TCP · UDP
Splits into segments, adds port numbers, sequence numbers.
Internet (Network) layer IP · ICMP
Adds source and destination IP addresses; routes the packet.
Link (Network access) layer Ethernet · Wi-Fi · MAC
Adds MAC addresses; turns it into a frame of bits on the wire.

3. Layer reference

What gets added (or removed) at each TCP/IP layer.

ApplicationHTTP, SMTP, FTP, DNS. The data the user actually wants. DNS turns curriculumexpert.co.uk into an IP address before sending.
Transport (TCP)Breaks data into segments, numbers them so they can be reassembled in order, adds source and destination port numbers (e.g. port 80 for HTTP).
Internet (IP)Adds source and destination IP addresses, making a packet. Routers use the destination IP to forward it hop by hop.
Link (Ethernet/Wi-Fi)Adds source and destination MAC addresses, making a frame, then transmits it as signals. MAC changes at every hop; IP stays the same end to end.
