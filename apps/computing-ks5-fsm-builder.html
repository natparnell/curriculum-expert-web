4. Why this matters

A finite state machine recognises a regular language: exactly the set of strings a regular expression can describe. Every accepting path through the machine is a way of matching the input.

From FSM to regular expression
  • A loop on a state means “zero or more”, like * in regex.
  • A choice of transitions means alternation, like |.
  • Strings (0+1)*0 are accepted by a two state machine ending on an accepting state after a 0.
The limit of finite state machines

An FSM has finite memory: it only ever knows which state it is in. So it cannot count without bound. The language { 0ⁿ1ⁿ } (equal numbers of 0s then 1s) is not regular: no FSM can recognise it. That job needs a more powerful model (a pushdown automaton or a Turing machine). This is the boundary between what an FSM can and cannot compute.

Keyboard and touch help

  • Add transition: press the transition button, click the source state, then the target. You will be asked for the symbol(s).
