    The input, process, output model

    Every product with embedded intelligence follows the same shape. A microcontroller is the processing brain in the middle.

    Input

    Sensors detect the world: a button press, light level, temperature.

    →

    Process

    The microcontroller runs your program: read, decide, set.

    →

    Output

    Actuators act on the world: an LED, a buzzer, a motor.

    Programmable components

    • A microcontroller is a tiny computer on a single chip. It has pins you connect components to. You program it once, and it runs that program every time it is powered.
    • An input pin reads a sensor. A sensor turns something physical (light, heat, a press) into a value the program can test.
    • An output pin drives an actuator: a component that does something, such as an LED that lights, a buzzer that sounds, or a motor that turns.
    • A control program is a sequence of steps. We can draw it as a flowchart or build it from blocks: start, read an input, make a decision (an if), set an output, then loop back to repeat.
    • The decision block is what makes a product intelligent: it lets the same hardware behave differently depending on what the sensors detect.
    • A loop means the program keeps checking the inputs forever, so the product responds the moment conditions change.

    Worked example: a burglar alarm

    • Input: a push button on pin D2 stands in for a door switch.
    • Output: a buzzer on pin D7.
    • Program: loop forever — read the button; if the button is pressed, set the buzzer ON, else set the buzzer OFF.

    The same shape builds a night-light (light sensor and LED), a counter (button and display), or a thermostat (temperature sensor and motor/fan). Change the sensor, the actuator and the decision; the structure stays the same.
