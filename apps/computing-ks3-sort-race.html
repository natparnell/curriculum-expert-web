Teacher notes and the big idea (tap to open)
  • Abstraction: a bar's height stands for a number's value. We hide the real data and reason about heights, the same idea as comparing variables in code.
  • Comparisons are the cost: both sorts swap and shuffle, but the fair measure of work is how many times each asks "is A greater than B?". That count is what we race.
  • The key result: on a nearly sorted list, insertion sort usually wins clearly, because it stops scanning as soon as an item is in place. On a reversed list both do a lot of work and the gap shrinks. Try both list types to show pupils that "best algorithm" depends on the data.
  • Tracing: use "Step once" with a slow speed to trace each comparison by hand, then check the live counter against pupils' tallies.
  • Stretch: ask pupils to design a list where bubble sort beats insertion sort. (It cannot: insertion never does more comparisons than bubble on the same data.)
