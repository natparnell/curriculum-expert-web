Little Man Computer 🧠

Teacher aim: students write and run LMC assembly and watch the von Neumann fetch-decode-execute cycle move data between the program counter, accumulator and mailboxes one micro-step at a time. Project it, single-step it, narrate the registers.

Assembly editor 3-digit machine, mailboxes 00-99

Editor ready. Type or load an example, then assemble.

MnemonicCodeModeMeaning
INP901registerRead next input into the accumulator.
OUT902registerCopy the accumulator to the output stream.
LDA xx5xxdirectLoad contents of mailbox xx into the accumulator.
STA xx3xxdirectStore the accumulator into mailbox xx.
ADD xx1xxdirectAdd contents of mailbox xx to the accumulator.
SUB xx2xxdirectSubtract contents of mailbox xx from the accumulator.
BRA xx6xxdirect (jump)Branch always: set program counter to xx.
BRZ xx7xxdirect (jump)Branch if accumulator is zero.
BRP xx8xxdirect (jump)Branch if accumulator is positive (zero or more).
HLT000registerHalt the processor.
DAT nndata labelReserve a mailbox holding the value n (default 0).

Addressing modes shown above: register/implied (INP, OUT, HLT need no operand), direct/absolute (LDA, STA, ADD, SUB read or write the named mailbox), and direct addresses used as a jump target by BRA, BRZ, BRP. LMC has no immediate or indirect mode, which is itself a useful discussion point.
