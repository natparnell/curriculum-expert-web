Key ideas (OCR J277, data representation)

Place value. An 8-bit number (a byte) has columns worth 128 64 32 16 8 4 2 1 from left (most significant bit, MSB) to right (least significant bit, LSB). The denary value is the sum of the place values that have a 1. With 8 bits you can store 2^8 = 256 different values, that is 0 to 255.

Hexadecimal. Base 16 uses digits 0 to 9 then A to F (A = 10, B = 11, C = 12, D = 13, E = 14, F = 15). One hex digit represents exactly 4 bits (a nibble), so one byte is always two hex digits. We use hex because it is far shorter and less error-prone than long binary strings, it maps cleanly to binary (each nibble to one digit), and it is widely used for memory addresses, colour codes and machine code.

Binary addition. Add column by column from the LSB. Rules: 0+0=0, 1+0=1, 1+1=10 (write 0, carry 1), 1+1+1=11 (write 1, carry 1). If a carry comes out of the leftmost column the answer needs a 9th bit, so it cannot fit in 8 bits: this is overflow.

Binary shifts. A logical shift moves every bit a number of places. A left shift by n multiplies the value by 2^n (a 0 fills each vacated position on the right). A right shift by n divides by 2^n and discards any fraction. Bits shifted off the end of the byte are lost, which can change the value unexpectedly if a 1 falls off.
