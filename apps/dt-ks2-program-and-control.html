    How program and control works

    The big idea in this strand of Design and Technology.

    Input

    An input is information coming in. A switch or a sensor tells the product what is happening around it.

    Example: a light sensor or a push switch.

    Process

    The process is the program: a sequence of instructions the product follows, one after another. It can wait, repeat, and use "if" to make a decision.

    Example: "if it is dark, turn the lamp on".

    Output

    An output is the thing the product does. The program switches outputs on and off using electrical systems like a series circuit.

    Example: a bulb, a buzzer or a motor.

    Program and control

    A product can respond to an input and control an output by following a program. The program is just a list of instructions in order. Wait holds for a time. Repeat runs a block of steps over and over. If only runs its steps when the sensor or switch is on. Change the program and run it again to see the effect: that is how you program, monitor and control a real product.

    Keyboard: every block and button works with Tab and Enter. In your program, + and − on a wait or repeat block change its number; the up and down arrows reorder steps. The light/dark sensor uses the Space key when focused on the input switch.
