How it works

The two binomial theorems

Positive integer n. The expansion terminates after n + 1 terms:

(a + b)n = Σ nCr an−r br,   r = 0 to n

where nCr = n! / (r!·(n−r)!). The coefficients are exactly row n of Pascal's triangle. This is on the formula sheet for Edexcel 9MA0 Paper 1.

Any n (the general binomial series). For |x| < 1 the series is infinite:

(1 + x)n = 1 + nx + n(n−1)2!x2 + n(n−1)(n−2)3!x3 + ...

Here n need not be a whole number, so there is no top row of Pascal and no factorial of n; the products in the numerators simply keep going.

Why validity matters: |x| < 1

When n is a positive integer the series stops, so it is exact for every x. When n is negative or fractional the series is infinite and only converges when |x| < 1. Outside that interval the terms grow without bound and the partial sums diverge: that is the runaway you see on the graph.

For (a + bx)n we first factor out a: (a + bx)n = an(1 + (b/a)x)n. The bracket is now in (1 + X)n form with X = (b/a)x, so the condition becomes |(b/a)x| < 1, i.e. |x| < |a/b|.

Using a few terms to approximate

To estimate something like √1.02, write it as (1 + 0.02)1/2. Because 0.02 is tiny, x2 = 0.0004 is already small and x3 is negligible, so two or three terms land very close to the true value. The smaller |x|, the faster the convergence: this is exactly why exam questions choose values like 1.01 or 0.98.
