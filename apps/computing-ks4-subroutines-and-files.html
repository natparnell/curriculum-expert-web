global frame procedure function

A new frame is pushed onto the stack each time a subroutine is called. The frame holds that call's parameters and local variables only. When the subroutine returns, its frame is popped, so those locals disappear. This is why a local variable cannot be seen outside its subroutine: that is scope.

The key ideas

Subroutine

A named block of code you can call from elsewhere. Writing it once and calling it many times reduces repetition, makes a program easier to read, and means a fix is made in one place only. This is why subroutines aid maintainability.

Procedure vs function

Both are subroutines. A procedure carries out a task but does not send a value back. A function returns a value with return, so its result can be stored or used in an expression, e.g. total = add(3, 4).

Parameters & return values

A parameter passes data into a subroutine when it is called, e.g. function area(w, h). A return value passes a result back out to the caller. The arguments you supply at the call site fill the parameters.

Local vs global & scope

A local variable exists only inside the subroutine that declares it; it lives in that call's stack frame and vanishes on return. A global variable is visible everywhere. Scope is the region of a program where a variable can be used. Prefer locals: they keep subroutines self-contained and easier to maintain.

Exam-style questions click to reveal

Q1State one difference between a procedure and a function.[1 mark]
Indicative answer
  • A function returns a value to the calling code, whereas a procedure does not return a value.
Q2Explain two reasons why a programmer uses subroutines.[4 marks]
Indicative answer
  • Reusability / less repetition: code written once can be called many times, so the program is shorter and there is no duplicated code.
  • Maintainability / easier to read: the program is broken into smaller named parts, so it is easier to understand, test and debug, and a change need only be made in one place.
Q3The subroutine below is written in OCR Exam Reference Language.
function triple(n)   return n * 3   endfunction
Identify the parameter, and state what the line result = triple(5) stores in result.[2 marks]
Indicative answer
  • The parameter is n.
  • result stores 15 (the value returned by the function when called with the argument 5).
Q4A program declares x as a local variable inside a function. Explain why x cannot be used in the main program.[2 marks]
Indicative answer
  • x has local scope, so it only exists inside the function (in that call's stack frame).
  • When the function returns, the frame is removed and the variable no longer exists, so the main program cannot access it.
Q5Write OCR Exam Reference Language to open the file scores.txt, read and print every line, then close it.[5 marks]
Indicative answer
1myFile = openRead("scores.txt")
2while NOT myFile.endOfFile()
3 print(myFile.readLine())
4endwhile
5myFile.close()

Mark points: open the named file to read; loop using endOfFile(); readLine() inside the loop; print the line; close the file.

Q6Using the runner above, a program calls add(a, b) from inside main. Describe what happens to the call stack when add is called and when it returns.[3 marks]
Indicative answer
  • When add is called, a new stack frame is pushed on top, holding its parameters a and b and any local variables.
  • Control passes into add; the stack depth increases by one.
  • When return runs, the frame is popped, control returns to main, and the returned value is used there.
