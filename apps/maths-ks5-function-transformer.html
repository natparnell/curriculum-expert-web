Function Transformer

Teacher aim: move sliders to translate, stretch and reflect a function. Read off the exact algebra, watch the domain and range update, and use the mapping diagram to see composite and inverse functions. Pure A-level: functions and transformations.

Graph y = a·f(b(x + c)) + d

Base f(x) Transformed

y = f(x)

Slide a control to transform the function. The yellow term shows what just changed.

Domain
all real x
Range
y ≥ 0

Controls

1

y → a·f(x). |a| > 1 stretches, 0 < |a| < 1 squashes, a < 0 reflects in the x-axis.

1

y → f(b·x). Scale factor 1/b. b < 0 reflects in the y-axis.

0

y → f(x + c). Replace x with x + c shifts left by c (note the sign).

0

y → f(x) + d. Shifts the whole curve up by d.

Composite view: an input x is fed through the transformed function. Drag the input slider and watch it map to an output.

Order the four steps to prove that the transformation y = f(x + c) + d maps the point (p, q) on y = f(x) to (p − c, q + d). Use the arrows or drag.

    Build the proof in logical order.
    How do I drive this with a keyboard?

    Tab to any slider, then use the left and right arrow keys to change each value by one step. The graph, algebra, domain and range update live and are announced. The graph itself can be focused to read its description.
