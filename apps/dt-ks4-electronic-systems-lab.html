The systems approach ▸

Every electronic product can be described as three connected stages, the systems approach. Designers work block by block so a complex product becomes manageable.

  • Input A sensor converts a physical quantity (light, heat, movement) into an electrical signal. The reading changes as conditions change.
  • Process A device decides what to do with the input. Here it is a comparator: it compares the sensor reading against a fixed threshold (set point) and gives a high or low output.
  • Output An actuator turns the decision into an effect we can see, hear or feel: light from an LED or lamp, sound from a buzzer, or movement from a motor.

Drawing the system as a block diagram (input box → process box → output box) is a key design tool in Eduqas GCSE D&T.

Input components: sensors ▸
  • Switch A simple input that makes or breaks a circuit (push-to-make, push-to-break, toggle). It gives a clean on or off signal.
  • Light-dependent resistor (LDR) Its resistance falls as light increases. In bright light resistance is low; in darkness it is high. Used in a potential divider to give a changing voltage to the process stage.
  • Thermistor A temperature-sensitive resistor. A common (NTC) thermistor has resistance that falls as temperature rises.
  • Variable resistor (potentiometer) A manually adjusted resistor, often used to set sensitivity or a reference voltage by hand.

The LDR and thermistor work in a potential divider: pairing the sensor with a fixed resistor turns its changing resistance into a changing voltage the comparator can read.

Process devices: comparator, transistor, microcontroller ▸
  • Comparator Compares the input voltage to a reference (the threshold set by a variable resistor). When the input crosses the set point the comparator switches its output high or low. This gives the system a clear on or off decision.
  • Transistor as a switch A small base current lets a much larger current flow to the output. It acts as an electronic switch and lets a tiny sensor signal control a high-current device such as a motor or lamp.
  • Programmable component (microcontroller) A single chip programmed by the designer, often using a flowchart. It reads inputs, makes decisions and controls outputs in software, so one chip can replace many fixed components and the behaviour can be changed by reprogramming. Try the Flowchart programmer tab.
Output components and protecting them ▸
  • LED A light-emitting diode. It must be fitted the correct way round and always needs a series resistor to limit the current, or it will draw too much current and be destroyed.
  • Buzzer Produces a sound output, useful for an alarm.
  • Lamp A filament or LED lamp for higher light output.
  • Motor Produces movement (rotation). It needs more current than a chip can supply directly, so a transistor drives it and a diode protects against back-EMF.

Protecting components: resistors limit current to safe values (for example a current-limiting resistor in series with an LED). Choosing the right resistor value is part of designing a reliable circuit.

Why program with a flowchart? ▸

A microcontroller is a programmable component: a single chip that can read inputs, make decisions and drive outputs. The designer writes the behaviour as software rather than wiring fixed components, so the same chip can become a night-light, an alarm or a timer just by changing the program.

At GCSE the program is usually planned as a flowchart using standard symbols. A decision (diamond) tests a condition and branches yes or no, which is how the chip reacts to its sensors. Step through your flowchart above to follow the path the chip would take for the current sensor reading.

  • Flexibility: reprogram instead of rebuild.
  • Fewer components: one chip replaces many process and timing parts.
  • Smarter behaviour: counting, timing and multiple decisions are easy in software.
