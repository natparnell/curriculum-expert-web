Visualising Data Structures

Teaching aim: build a secure mental model of how the core abstract data types behave by watching their operations run step by step. Push and pop a stack and see the stack pointer move; enqueue and dequeue a circular queue and watch the rear wrap round; build a binary search tree and run each traversal; and hash a sequence of keys into a hash table, resolving collisions by chaining.

About the stack
Specification reference: data structures

OCR H446 section 1.4.2 requires learners to know how the following structures are used, and to be able to read, write, follow and trace algorithms that operate on them:

  • Stacks (LIFO): push, pop, peek, the stack pointer, overflow and underflow.
  • Queues (FIFO): linear and circular queues, enqueue and dequeue, front and rear pointers.
  • Linked lists: nodes of data and a pointer, traversal, insertion and deletion.
  • Trees: binary search trees, insertion, and pre-order, in-order and post-order traversals.
  • Hash tables: a hash function mapping a key to an index, collisions and a resolution method (chaining or open addressing).

These are abstract data types: the specification cares about behaviour, not the underlying implementation, which may use a static array or dynamic nodes.
