Teacher and pupil notes: ADTs, Dijkstra vs A*, and Big-O

Graph Pathfinder makes one abstract data type visible: the min-priority queue. Both algorithms keep a frontier of discovered-but-not-yet-settled nodes ordered by priority, and repeatedly pop the cheapest one. This is the engine of shortest-path search.

Dijkstra's algorithm

Priority is the distance from the start, written g(n). Dijkstra has no idea where the goal is, so it expands outward like a wavefront. It is guaranteed to find the shortest path when all edge weights are non-negative.

A* search

Priority is f(n) = g(n) + h(n), where h(n) is a heuristic estimate of the remaining distance to the goal (here, straight-line distance on the canvas). Because A* is pulled towards the goal, it usually settles fewer nodes than Dijkstra. With an admissible heuristic (one that never overestimates) A* still returns the true shortest path. Compare the Nodes settled metric for the same start and goal to see the saving.

Why a heap, and the Big-O

A binary heap (the usual priority-queue implementation) does insert and extract-min in O(log n). With V vertices and E edges, Dijkstra with a binary heap runs in O((V + E) log V). A linear-scan queue would be O(V²), which is why the choice of ADT changes the complexity class.
